# step 4 — 스킬 생성 (템플릿 기반 즉석 구성) + 오케스트레이터

> 반복 워크플로("어떻게")를 도구+도메인+discovery 근거로 직접 설계·생성한다.
> 상시 비용은 description만 — 본문은 트리거 시 로드. 형식은 `templates/SKILL.md.tmpl` 주석이 강제.

## 할 일 (대상 프로젝트 `.claude/skills/`에 생성)
1. **목록 제안**: step3 에이전트별로 반복될 핵심 워크플로를 스킬 후보로 도출해
   이름·담당 에이전트·근거를 표로 제안. TDD 재현-수정(버그픽스)류는 스택 무관하게 항상 포함 권장.
2. ◀**승인 게이트**: 목록 확정 후 생성.
3. **생성** (한 번에 1개씩): `templates/SKILL.md.tmpl` 골격+주석 지시대로.
   - description은 pushy + near-miss 3요소. 명령은 `kb/tooling-matrix.md` 치환, 패턴 근거는 discovery의 실제 코드.
   - 구현 스킬 절차는 **RED(실패 테스트 커밋) → GREEN(구현, 테스트 수정 혼입 금지) → REFACTOR** 유지.
   - 템플릿의 `<!-- 작성 지침 -->` 주석은 채운 뒤 **산출물에서 제거**한다(생성용 지침, 런타임 불필요).
   - 본문 ≤500줄, 도구별 변형은 references/로. 앱/인프라 혼합 금지. `examples/`는 픽스처 — 참조 금지.
4. **오케스트레이터 스킬 1개**: 팀 전체 흐름. 필요한 에이전트만 온디맨드 스폰(전부 동시 로드 금지).

## 오케스트레이터 기준
- 팀 모드: TeamCreate → TaskCreate(의존: 구현→테스트→배포plan) → 자체 조율 → 취합.
  서브 모드: Agent(…, run_in_background) 병렬 → 수집.
- 데이터 전달: **태스크(조율) + 파일(산출물) + 메시지(소통)**. 중간 산출물 `_workspace/{NN}_{agent}_{artifact}.{ext}`,
  최종만 사용자 경로, 중간 파일은 감사용 보존.
- 에러: 공통(1회 재시도→누락 명시·증거 보존)은 `kb/design-principles.md` §5. 특화:
  terraform plan 예상 밖 차이→중단·사람 제시·apply 금지 / 범위 침범→거부+담당 위임 / 빌드환경→matrix 재확인 후 1회 재시도.
- 팀 크기: 소(작업5~10)=2~3명, 중=3~5, 대=5~7. **집중된 3명 > 산만한 5명.**
- 테스트 시나리오 포함: 정상 1 + 에러 1+(테스트 실패 폴백, apply 차단).
